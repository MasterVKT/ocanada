**1. Identité de la Marque**

**1.1 Positionnement**

Ô Canada est une entreprise locale basée à Douala, Cameroun.
L\'application de gestion RH doit véhiculer les valeurs suivantes :
professionnalisme, fiabilité, modernité et adaptation au contexte local
camerounais.

L\'identité visuelle de l\'application s\'inspire du nom de
l\'entreprise (référence canadienne) tout en s\'ancrant dans un contexte
africain et professionnel. Elle doit rester sobre, lisible et efficace
pour des utilisateurs de niveaux différents.

**1.2 Principes directeurs**

-   **:** L\'interface privilégie la lisibilité et la hiérarchie
    visuelle. Chaque élément a une fonction claire.

-   **:** Tous les composants respectent les mêmes règles visuelles sur
    l\'ensemble des pages.

-   **:** Contrastes suffisants, tailles de texte adaptées, navigation
    intuitive.

-   **:** L\'utilisateur trouve et exécute ses actions en un minimum de
    clics.

-   **:** Interface en français, formats de date et heure camerounais,
    devise XAF.

**1.3 Tone of Voice visuel**

Professionnel sans être froid --- L\'application s\'adresse à des
professionnels mais dans un contexte humain (RH). Le rouge primaire
apporte énergie et clarté d\'action, le gris foncé garantit le sérieux,
et les couleurs sémantiques guident l\'utilisateur sans l\'agresser.

**3.3 Règles typographiques**

-   **:** line-height: 1.6 pour le corps de texte standard, 1.4 pour les
    tableaux et listes compactes.

-   **:** letter-spacing: 0.025em pour les titres en majuscules (badges,
    labels ALL CAPS).

-   **:** Utiliser CSS text-transform: uppercase uniquement pour les
    badges de statut et labels courts. Ne jamais écrire en majuscules
    dans le HTML.

-   **:** Pour les colonnes de tableau à largeur fixe : text-overflow:
    ellipsis avec overflow: hidden et white-space: nowrap. Toujours
    associé à un title=\"\...\" HTML pour l\'info-bulle au survol.

-   **:** Les colonnes numériques (heures, durées, montants) sont
    alignées à droite (text-align: right).

Les cartes d\'indicateurs clés du tableau de bord administrateur suivent

-   **:** Icône Bootstrap Icons en position absolue en haut à droite de

**6. Navigation et Mise en Page**

**6.1 Structure générale des pages authentifiées**

Toutes les pages (hors kiosque et login) partagent la même structure de

-   **:** Hauteur 60px. Fond blanc. Ombre légère box-shadow. À gauche :

-   **:** Hauteur 40px. Fond blanc. Texte centré : \"Ô Canada RH ---
    v1.0 \| Douala, Cameroun\". Font-size 0.75rem, couleur muted.

**6.2 Comportement responsive de la sidebar**

-   **:** Sidebar toujours visible, fixe. Contenu décalé à droite de
    250px.

-   **:** Sidebar rétractée par défaut (width: 0, overflow: hidden).
    Toggle button visible dans la topbar. Au clic : sidebar s\'ouvre en
    overlay sur le contenu (position: fixed, z-index élevé).

-   **:** Même comportement que tablette. Les cartes KPI passent en
    colonne unique. Les tableaux deviennent scrollables horizontalement.

**6.3 Page de connexion**

Page autonome sans sidebar ni topbar. Mise en page centrée verticalement
et horizontalement (flexbox). Fond dégradé diagonal de #C41230 à
#8B0D21. Carte de connexion (card) blanche, largeur max 420px,
border-radius 12px, shadow-lg.

**6.4 Page kiosque**

Page autonome plein écran (100vw × 100vh). Fond #1A365D. Pas de menu,
pas de navigation. Trois zones verticales : en-tête (logo + date +
horloge), centre (formulaire de pointage), bas (message instructif).

-   **:** p-3 (16px) sur mobile, p-4 (24px) sur desktop.

-   **:** gap-3 (16px) dans la grille row.

-   **:** mb-4 (24px) entre les sections d\'une page.

-   **:** Padding 24px sur tous les côtés (px-3 py-3 sur mobile, p-4 sur
    desktop).

**7.3 Résolutions cibles**

-   **:** 1280 × 768 pixels (desktop, comme spécifié dans les SFD).

-   **:** 1366 × 768 ou 1920 × 1080 pixels.

-   **:** 768 × 1024 pixels (mode portrait et paysage).

-   **:** Conçu pour un écran de bureau standard (≥ 1024px). Optimisé
    pour un usage sans clavier tactile.

**8. Animations et Interactions**

**8.1 Principes d\'animation**

-   **:** 300ms pour les transitions d\'interface (ouverture modale,
    tooltip). 500ms pour les animations d\'entrée de page.

-   **:** ease-in-out pour les transitions, ease-out pour les
    apparitions d\'éléments.

-   **:** Toutes les animations respectent prefers-reduced-motion : si
    cette préférence est active, les animations sont désactivées.

**8.2 Interactions définies**

  -------------------------------------------------------------------------

**10. Règles d\'Application et Bonnes Pratiques**

**10.1 Règles d\'utilisation des couleurs**

-   Ne jamais utiliser le rouge primaire (#C41230) pour les messages
    d\'erreur (réservé aux actions, pas aux alertes). Utiliser #E53E3E
    pour les erreurs.

-   Le fond de page (#F7FAFC) ne doit jamais être utilisé comme fond de
    carte --- uniquement #FFFFFF ou #EDF2F7.

-   Le contraste texte/fond doit toujours respecter le ratio WCAG AA
    minimum (4.5:1 pour le texte normal, 3:1 pour les grands textes).

-   Les couleurs sémantiques (succès, danger, avertissement, info) sont
    réservées à leurs contextes définis. Ne pas les utiliser pour la
    décoration.

**10.2 Règles de contenu**

-   Tous les libellés, labels et textes de l\'interface sont en français
    standard camerounais. Aucune abréviation non définie.

-   Les dates sont toujours au format JJ/MM/AAAA. Les heures au format
    HH:MM (24h). La devise est affichée \"XAF\" ou \"FCFA\".

-   Les nombres décimaux utilisent la virgule comme séparateur (ex :
    \"18,5 jours\"). Les milliers utilisent l\'espace insécable (ex :
    \"1 250 000 XAF\").

-   Les messages d\'erreur sont toujours constructifs (\"Veuillez saisir
    un email valide\") et jamais techniques (\"NULL constraint
    violation\").

**10.3 Accessibilité minimale**

-   Tout élément interactif (bouton, lien, input) doit avoir un attribut
    aria-label si son contenu n\'est pas du texte lisible (ex : boutons
    icônes seules).

-   Les champs de formulaire doivent être associés à leur label via
    l\'attribut for/id ou en incluant le champ dans le label.

-   Les images décoratives doivent avoir alt=\"\" (chaîne vide). Les
    images informatives doivent avoir un alt descriptif.

-   La navigation clavier doit être fonctionnelle sur tous les
    composants interactifs (tabindex, focus visible).

**10.4 Performance**

-   Les images de profil employés sont affichées en miniature (max
    40×40px dans les tableaux, 80×80px dans les fiches). Le fichier
    source peut être plus grand mais l\'affichage est contraint par CSS.

-   Les icônes Bootstrap Icons sont chargées depuis CDN. Ne pas les
    charger en tant qu\'images SVG individuelles.

-   La police Inter est chargée avec display=swap pour éviter le FOIT
    (Flash of Invisible Text).

-   Le CSS personnalisé (ocanada.css) est minifié pour la production.
